In the previous lesson, we learned what time complexity is.

We also learned that time complexity tells us how many steps an algorithm takes as the input size grows.

But that raises an important question.

How do we actually count those steps?

We use a simple technique called the Frequency Count Method.

Don't let the name sound complicated.

The idea is actually very simple.

We count the number of steps a program takes to execute.

Let's see how it works with an example.

Our Assumption

To keep things simple, we'll make one assumption throughout this lesson.

Every simple statement takes 1 step.

For example,

int x = 10;

takes 1 step.

Similarly,

sum += x;

also takes 1 step.

So instead of worrying about how much real time a program takes, we'll simply count how many steps it performs.

After a few examples, you'll see that the Frequency Count Method is mostly just careful counting.

Example 1 — Statements That Execute Once

Let's begin with a simple program.

void greet() {
    cout << "Hello";
    cout << "Welcome";
    cout << "Have a great day!";
}

Now let's count how many times each statement executes.

Statement Frequency
cout << "Hello"; 1
cout << "Welcome"; 1
cout << "Have a great day!"; 1

Since every statement takes 1 step, the total number of steps is

1 + 1 + 1 = 3

We write this as

f(n) = 3

This is called the time function.

  • f(n) represents the total number of steps.
  • n represents the size of the input.

In this example, the algorithm always performs 3 steps, no matter what. So f(n) = 3.

What If a Statement Executes Multiple Times?

The previous example was easy because every statement executed only once.

But most real programs contain loops, where the same statement executes again and again.

Instead of counting a statement once, we'll count how many times the loop executes it.

Example 2 — A Single Loop

Consider the following program.

int sum = 0;

for (int i = 0; i < n; i++) {
    sum += i;
}

Let's first assume n = 5. That means the loop runs 5 times.

Now let's count how many times each statement executes.

Statement Frequency (when n = 5)
sum = 0; 1
sum += i; 5

Adding the frequencies together gives us f(5) = 1 + 5 = 6.

Of course, n won't always be 5. It could be 10, 100, or even 1,000.

Instead of writing a different answer every time, we simply replace 5 with n.

Statement Frequency
sum = 0; 1
sum += i; n

So the time function becomes

f(n) = n + 1

What Happens With Nested Loops?

Counting a single loop isn't too difficult.

But what happens when one loop is placed inside another?

In that case, we don't add the number of executions. We multiply them.

Example 3 — Nested Loops

for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        cout << "*";
    }
}

Let's assume n = 3.

The outer loop runs 3 times. For every iteration of the outer loop, the inner loop also runs 3 times.

Outer Loop Iteration Inner Loop Runs
1 3
2 3
3 3

So the statement cout << "*"; executes 3 × 3 = 9 times.

Therefore, f(3) = 9.

Now let's replace 3 with n. The time function becomes

f(n) = n × n = n²

Remember
Whenever one loop is inside another loop, we multiply the number of times they execute.

What Happens When the Value Doubles?

So far, every loop has increased by 1 during each iteration.

But not every loop works that way.

Sometimes, the value doubles instead. When that happens, the loop runs much fewer times.

Example 4 — Loops That Double Each Iteration

for (int i = 1; i < n; i *= 2) {
    cout << i;
}

Let's assume n = 16. The loop starts with i = 1. After every iteration, i is multiplied by 2.

Iteration Value of i
1 1
2 2
3 4
4 8

After the fourth iteration, i becomes 16. Since the loop only runs while i < n, the loop stops. That means cout << i; executes 4 times.

Let's try a few more values of n.

Value of n Loop Runs
8 3
16 4
32 5
64 6

Notice the pattern. Every time n doubles, the loop runs only one extra time.

If you've learned logarithms in school, these numbers might look familiar.

  • log₂ 8 = 3
  • log₂ 16 = 4
  • log₂ 32 = 5
  • log₂ 64 = 6

These are exactly the same values we got by counting the loop iterations.

So the time function becomes

f(n) = log₂ n

You'll often see it written simply as f(n) = log n, because in time complexity the base 2 is usually omitted.

Remember
When a loop doubles its value every iteration, the number of executions grows as log n.

Key Takeaways

Here's a quick summary of everything you learned in this lesson.

  • The Frequency Count Method calculates the total number of steps by counting how many times each statement executes.
  • We assume every simple statement takes 1 step.
  • To find the total number of steps, add the frequency of all the statements.
  • A statement outside a loop usually executes once.
  • A statement inside a loop that runs n times executes n times.
  • When one loop is inside another, multiply their frequencies, giving n².
  • When a loop doubles its value every iteration, the number of executions grows as log n.
  • The final expression that represents the total number of steps is called the time function, written as f(n).
